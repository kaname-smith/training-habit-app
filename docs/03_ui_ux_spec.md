# 03 UI/UX Spec — GUI設計仕様

## 1. UIコンセプト

**「試験期間でも開ける、静かなトレーニングアプリ」**

一般的な筋トレアプリのような強い煽り、過度な炎、連続記録の圧迫感は避ける。勉強中のユーザーが、夜に部屋で開いても疲れないUIにする。

## 2. デザイン原則

1. スマホ第一
2. 1画面1目的
3. 今日やることが3秒以内に分かる
4. 主要アクションは親指で押しやすくする
5. 休息・10分版を肯定的に扱う
6. 色は落ち着いたものを基調にする
7. 数字は大きく、説明文は短く
8. 失敗扱いの文言を避ける

## 3. 推奨ビジュアル方向

- ベース：白〜薄いグレー、または落ち着いたダークモード
- アクセント：青、緑、紫のいずれか。過度な赤は避ける
- カードUI中心
- 角丸、十分な余白
- アイコンは線画系
- 運動中画面は視認性最優先

## 4. 主要画面

### 4.1 初期設定画面

目的：ユーザーの状態を把握し、今日から始められる状態にする。

要素：

- アプリ名
- 短い説明
- 体重入力
- 運動経験：初心者 / 再開 / 継続中
- 試験モードON/OFF。初期値ON
- プロテイン1杯あたりのタンパク質量。初期値20g
- 「今日から始める」ボタン

文言例：

```text
7月は助走で十分です。試験勉強を壊さず、8月に向けて身体を起こしましょう。
```

### 4.2 今日のダッシュボード

目的：今日やることを即決できる。

ファーストビュー：

- 今日の日付
- 今日のおすすめ：筋トレA / 筋トレB / 10分版 / 休息
- 所要時間
- 主ボタン：「今日のメニューを始める」
- 副ボタン：「10分版にする」「今日は休息にする」

追加情報カード：

- 今月の実施回数
- 今週の実施回数
- タンパク質目標
- 睡眠・疲労チェック

### 4.3 トレーニング実行画面

目的：迷わず実行できる。

要素：

- 種目名
- セット数
- 回数/秒数
- フォーム注意1文
- セット完了チェック
- 次の種目ボタン
- 軽量版へ変更
- 中止して休息記録

運動中は長文説明を避ける。

### 4.4 トレーニング完了画面

目的：完了感と次回への接続。

要素：

- 「完了」表示
- 今日の実施内容
- 体感強度：軽い / ちょうどよい / きつい
- 次回の提案
- プロテイン記録への導線

文言例：

```text
今日はこれで十分です。試験月の目的は、途切れさせないことです。
```

### 4.5 記録・カレンダー画面

目的：7月の継続状況を可視化する。

表示：

- 月間カレンダー
- A/B/10分/休息/未記録を色またはアイコンで区別
- 実施回数
- 10分版を含む継続日
- 週ごとの合計

重要：休息日は空白ではなく、意図的な休息として表示する。

### 4.6 プロテイン・栄養画面

目的：タンパク質不足を簡単に把握する。

要素：

- 今日のタンパク質目標
- 今日の推定摂取量
- プロテイン杯数ボタン：0 / 1 / 2
- 食事由来の概算入力
- 体重別目標説明

過度なカロリー管理はMVPでは不要。

### 4.7 設定画面

要素：

- 体重
- プロテイン1杯量
- 試験モード
- 通知時刻
- データエクスポート
- データリセット
- アプリの注意事項

## 5. ナビゲーション

スマホでは下部タブを推奨。

| タブ | 内容 |
|---|---|
| 今日 | ダッシュボード |
| 実行 | トレーニング |
| 記録 | カレンダー・履歴 |
| 栄養 | プロテイン |
| 設定 | 設定 |

## 6. 状態別メッセージ

### 通常日

```text
今日は25分だけ。余裕を残して終わりましょう。
```

### 試験前日

```text
今日は10分版で十分です。勉強と睡眠を優先しましょう。
```

### 睡眠不足

```text
睡眠不足の日は、筋トレより回復を優先して問題ありません。
```

### 休息選択時

```text
休息も計画の一部です。明日戻れば大丈夫です。
```

## 7. アクセシビリティ

- 文字サイズは小さくしすぎない
- コントラストを確保する
- 色だけで状態を表現しない
- ボタンにはテキストラベルを付ける
- キーボード操作も最低限可能にする
- `prefers-reduced-motion` を尊重する

## 8. GUI完成度の判断基準

- スマホで片手操作できる
- 初見でも今日やることが分かる
- 休息・10分版が選びやすい
- 記録が楽しいが、圧迫感がない
- フォーム注意が短く実用的
- 情報の密度が高すぎない
- 7月の試験月という文脈がUIから伝わる
