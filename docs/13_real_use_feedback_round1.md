# 13 Real-use Feedback Round 1

## 1. 位置づけ

GitHub Pages上のStudyFit Primerをスマートフォンで実利用した結果、基本レイアウトは洗練されており、継続する価値があることを確認した。一方、実行支援、説明責任、栄養データ構造、利用者自身のフィードバック循環、学習管理への拡張に改善余地がある。

## 2. 維持すべき長所

- スマートフォン向けレイアウト
- 今日やることが分かるカード構成
- 10分版・休息を否定しない心理設計
- localStorage中心のプライバシー重視
- export/importによるデータ保全
- GitHub Pagesによる同一URL運用

UIを全面刷新しない。既存デザインシステムを継承し、必要な支援を段階的に追加する。

## 3. トレーニング実行時の改善点

### 3.1 時間種目

プランク等について、指定秒数を測れるカウントダウンタイマーが必要。

必要機能：

- 開始、一時停止、再開、リセット
- 残り時間表示
- 終了時の音・振動は設定可能
- バックグラウンド移行から復帰した場合も実時間に基づいて補正
- セット間休憩タイマー
- 音を出せない環境に配慮した視覚通知

### 3.2 回数種目

腕立て伏せ等について、反復のペースを示すテンポガイドが必要。

初期プリセット：

- 速い：2秒/回
- 中：3秒/回
- 遅い：5秒/回

ただし、これらは運動効果を断定する処方ではなく、一定の動作速度を保つための実行補助として扱う。将来はユーザーが秒数を調整可能にする。

表示案：

- 円形メトロノーム
- 「下ろす」「止める」「上げる」のフェーズ表示
- 音、振動、画面点滅のON/OFF
- 回数カウントは手動を基本とし、自動認識は対象外

## 4. 種目解説の改善点

種目名だけでは動作や対象部位が不明な場合がある。各種目に次を持たせる。

- 何を鍛える種目か
- 主働筋・補助筋
- 開始姿勢
- 動作手順
- 呼吸
- よくある誤り
- 簡単な代替種目
- 難しい発展種目
- 痛みが出た場合の中止注意

実行画面には1〜2行の要点を表示し、詳細は展開式にする。

## 5. 根拠説明の改善点

トレーニング頻度、強度、タンパク質目標などについて、根拠をまとめた独立ページが必要。

ページ名案：`根拠` / `Evidence`

単なるリンク集ではなく、各推奨事項について以下を説明する。

- アプリが採用しているルール
- 根拠資料
- 対象者
- 何が分かっているか
- 何が分かっていないか
- このアプリでの簡略化
- 最終確認日

## 6. 栄養記録の改善点

現行の「プロテイン1杯」を一般化する。

- 製品カタログを事前登録
- 商品名、メーカー、1食量、単位、タンパク質量を保持
- 記録時は商品と摂取量を選択
- 商品ごとのスプーン量・1杯量の違いに対応
- 記録時に新商品を追加可能
- 食事由来タンパク質を小数点第一位まで入力
- 将来、炭水化物、脂質、エネルギー等を追加可能なデータモデル

## 7. フィードバック循環

利用者が気づきを即座に残せるページを追加する。

入力区分：

- 不具合
- 分かりにくい
- 改善案
- 新機能案
- 根拠への疑問
- その他

データはlocalStorage保存とし、export対象に含める。外部送信はしない。

## 8. 学習管理への拡張

StudyFitの名称に合わせ、身体管理と学習管理を同じアプリ内で扱う。

ただし、現在は試験日・範囲・資料・到達度が不明確であるため、最初の機能は「予定を組むこと」ではなく「前提情報を収集して不確実性を減らすこと」とする。
