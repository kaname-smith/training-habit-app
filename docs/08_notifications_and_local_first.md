# 08 Notifications and Local-first — 通知・ローカル保存方針

## 1. ローカルファースト方針

MVPでは、すべての記録を端末内に保存する。

理由：

- 健康・体重・習慣データは個人性が高い。
- 初期開発では認証やサーバー管理を避ける方が安全。
- ユーザーがすぐ使える。
- Claude CodeによるMVP実装が明確になる。

## 2. 保存対象

- プロフィール
- 体重
- 試験モード
- トレーニング記録
- プロテイン記録
- 疲労チェック
- 設定

## 3. 保存しないもの

- 位置情報
- 連絡先
- 外部アカウント
- 医療診断情報
- 写真・動画

## 4. 通知方針

PWA通知は Phase 2 でよい。

MVPでは、アプリ内に「おすすめ時刻」を表示する。

通知文言例：

```text
今日は10分だけで十分です。勉強の合間に軽く動きましょう。
```

```text
試験前日は軽めでOK。睡眠を優先しましょう。
```

## 5. 通知設定

- 通知ON/OFF
- 推奨時刻：朝 / 夕方 / 夜
- 試験前日は軽量通知
- 休息推奨通知

## 6. データエクスポート

ユーザーがJSONで記録を出せるとよい。

MVPでは設定画面に「データを書き出す」を置く。

## 7. データリセット

リセット時は確認を2段階にする。

```text
本当にすべての記録を削除しますか？この操作は元に戻せません。
```
