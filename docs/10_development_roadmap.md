# 10 Development Roadmap — 実装ロードマップ

## Phase 0 — 読解・計画

Claude Code は最初にコードを書かず、仕様を読んで計画を出す。

成果物：

- 画面構成
- 技術構成
- 実装順序
- テスト方針
- リスク

## Phase 1 — MVP

目的：7月中に使える最小アプリ。

実装範囲：

- Vite + React + TypeScript プロジェクト
- 基本レイアウト
- 初期設定
- 今日のダッシュボード
- A/B/10分版/休息メニュー
- トレーニング実行
- ローカル保存
- 記録カレンダー
- プロテイン計算・記録
- 設定
- 最低限のテスト

完了条件：`docs/09_testing_acceptance.md` のMVP項目を満たす。

## Phase 2 — GUI強化

目的：毎日起動したくなるUIへ改善。

実装範囲：

- より洗練されたカードUI
- 状態ごとのメッセージ改善
- 空状態の改善
- 完了画面の演出
- レスポンシブ最適化
- ダークモード
- PWA manifest
- 可能なら簡易通知

## Phase 3 — 8月本格強化モード

目的：夏休みの本格トレーニングに移行。

実装範囲：

- 週4回モード
- 強化版A/B
- 種目ごとの負荷記録
- ダンベル/ジム種目を追加可能なデータ構造
- RIR記録
- 進歩グラフ

## Phase 4 — データ・分析

目的：長期継続を支える。

実装範囲：

- JSON/CSVエクスポート
- 週次レビュー
- 種目別進捗
- タンパク質達成率
- 睡眠・疲労との関係表示

## Phase 5 — 将来拡張

- IndexedDB化
- クラウド同期
- スマホネイティブ化
- Health連携
- フォーム動画参照
- 食事テンプレート

## 実装時の優先判断

迷ったら以下を優先する。

1. 7月中に使えること
2. 試験勉強を邪魔しないこと
3. スマホで使いやすいこと
4. データが消えないこと
5. 8月に拡張しやすいこと

## Claude Code への作業単位

推奨する依頼順：

1. `prompts/00_bootstrap_prompt.md`
2. `prompts/01_implementation_prompt.md`
3. `prompts/02_review_prompt.md`
4. GUI改善依頼
5. テスト強化依頼
6. 8月モード追加依頼
