# 09 Testing and Acceptance Criteria — テスト・受け入れ条件

## 1. 完了条件

MVPは、以下を満たしたら完了とする。

## 2. 機能受け入れ条件

### 初期設定

- [ ] 初回起動時に初期設定画面が表示される
- [ ] 体重を入力できる
- [ ] 運動経験を選べる
- [ ] 試験モードが初期ONである
- [ ] プロテイン1杯量を設定できる
- [ ] 設定後、今日の画面に遷移する
- [ ] 再起動しても設定が保持される

### 今日のダッシュボード

- [ ] 今日の日付が表示される
- [ ] 推奨メニューが表示される
- [ ] 推奨理由が表示される
- [ ] 「今日のメニューを始める」が押せる
- [ ] 10分版へ変更できる
- [ ] 休息に変更できる
- [ ] タンパク質目標が表示される

### トレーニング実行

- [ ] Aメニューを実行できる
- [ ] Bメニューを実行できる
- [ ] 10分版を実行できる
- [ ] 各種目の回数・セットが表示される
- [ ] セット完了を記録できる
- [ ] フォーム注意が表示される
- [ ] 完了時にログが保存される
- [ ] 軽量版に切り替えられる
- [ ] 中止して休息記録にできる

### 記録

- [ ] カレンダーに実施状況が表示される
- [ ] A/B/10分/休息が区別される
- [ ] 今月の実施回数が表示される
- [ ] アプリ再起動後も記録が保持される

### 栄養

- [ ] 体重からタンパク質目標が計算される
- [ ] プロテイン杯数を記録できる
- [ ] 食事由来タンパク質を概算入力できる
- [ ] 合計摂取量が表示される
- [ ] 再起動後も記録が保持される

### 設定

- [ ] 体重を変更できる
- [ ] プロテイン1杯量を変更できる
- [ ] 試験モードを切り替えられる
- [ ] データをリセットできる
- [ ] 安全上の注意が表示される

## 3. UI受け入れ条件

- [ ] スマホ幅で破綻しない
- [ ] 下部タブまたは同等の簡単なナビゲーションがある
- [ ] 今日やることがファーストビューで分かる
- [ ] 主要ボタンが押しやすい
- [ ] 休息・10分版がネガティブに扱われていない
- [ ] 色だけに依存しない状態表示になっている
- [ ] 長文説明が画面を圧迫していない

## 4. 品質受け入れ条件

- [ ] TypeScriptの型エラーがない
- [ ] lintエラーがない
- [ ] buildできる
- [ ] 主要ロジックにユニットテストがある
- [ ] タンパク質計算のテストがある
- [ ] スケジュール生成のテストがある
- [ ] ローカル保存のテストまたは手動確認がある

## 5. E2Eシナリオ

### シナリオ1：初回利用

```text
初回起動
→ 体重60kgで設定
→ 今日の画面
→ 10分版を開始
→ すべて完了
→ プロテイン1杯を記録
→ 記録画面で反映確認
```

### シナリオ2：疲労時

```text
今日の画面
→ 疲労チェックで「疲れている」
→ 10分版が推奨される
→ 休息に変更
→ カレンダーに休息として表示
```

### シナリオ3：データ保持

```text
トレーニング記録
→ ブラウザ更新
→ 記録が残っている
```

## 6. リリース前チェック

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

上記コマンドが存在しない場合は、実装中に追加する。
