# 07 Habit Logic — 習慣化ロジック

## 1. 基本思想

このアプリの習慣化ロジックは、ユーザーを追い詰めない。

一般的なストリーク型アプリは、途切れた瞬間にモチベーションが落ちる。試験月にはそれが逆効果になりやすい。したがって、このアプリでは「連続記録」よりも「復帰のしやすさ」を重視する。

## 2. 記録の種類

| 状態 | 意味 | UI表示 |
|---|---|---|
| completed | 通常メニュー完了 | 強めに肯定 |
| short_done | 10分版完了 | completedと同等に肯定 |
| rest | 意図的な休息 | 計画の一部として表示 |
| skipped | 未記録 | 失敗とは表示しない |
| rescheduled | 予定変更 | 柔軟な調整として表示 |

## 3. 推奨判定

優先順位：

1. 体調不良 → 休息
2. 睡眠不足 → 10分版または休息
3. 試験前日 → 10分版
4. 前回から48時間以上空いている → A/Bの次メニュー
5. 前日筋トレ済み → 休息または散歩
6. 週3回未満 → A/B推奨
7. 週3回達成済み → 休息または散歩

## 4. 疲労チェック

ユーザーに毎回細かく入力させすぎない。

簡易入力：

- 睡眠：十分 / 少なめ / かなり少ない
- 疲労：軽い / 普通 / 重い
- 明日試験：はい / いいえ

判定例：

```ts
if (hasPainOrSickness) return 'rest';
if (sleep === 'very_low') return 'rest';
if (sleep === 'low' || fatigue === 'heavy') return 'short';
if (examTomorrow) return 'short';
return scheduledWorkout;
```

## 5. メッセージング

### 通常完了

```text
完了です。余裕を残して終われたなら、今月は理想的です。
```

### 10分版完了

```text
10分版でつなげました。試験月ではこれも完全に成功です。
```

### 休息

```text
休息も計画の一部です。睡眠と勉強を優先しましょう。
```

### 未記録翌日

```text
昨日できなくても問題ありません。今日は10分だけ戻れば十分です。
```

## 6. 実績表示

推奨指標：

- 今月の実施回数
- 今週の実施回数
- 10分版を含む継続回数
- 休息を含む計画遵守日数
- 8月に向けた準備度

避ける指標：

- 失敗日数
- サボり日数
- 厳密な連続日数だけを強調するUI

## 7. 8月への移行判定

7/31時点で以下なら「8月本格モード解放」を表示する。

- 7月の筋トレ回数が8回以上
- または、10分版を含む実施回数が10回以上
- または、週2回以上を2週間継続

文言：

```text
7月の助走は十分です。8月からは少しずつ負荷を上げられます。
```
