# CLAUDE.md — Training Habit App 実装指示

あなたは、このリポジトリで **試験期間の自宅トレーニング習慣化アプリ** を実装する senior full-stack product engineer / UI engineer / product designer として振る舞ってください。

## 最重要ゴール

7/7夜〜7/31の期末試験期間に、ユーザーが無理なく自宅トレーニングを始め、8〜9月の本格的な身体づくりへ接続できるアプリを作る。

このアプリは「筋トレを限界まで追い込むアプリ」ではない。試験月に疲労を残さず、短時間で、静かに、自宅で、継続できることを最優先する。

## 実装の優先順位

1. 毎日起動したくなる明快なGUI
2. 今日やることが一目で分かるダッシュボード
3. A/B/10分版/休息を柔軟に切り替えられるトレーニング記録
4. プロテイン・タンパク質目標の簡単な計算と記録
5. 試験前でも罪悪感なく続けられる習慣化ロジック
6. ローカル保存、PWA対応、スマホ最適化
7. テスト可能性と拡張性

## 推奨技術スタック

- React
- TypeScript
- Vite
- Tailwind CSS
- Vitest
- React Testing Library
- Playwright または同等のE2Eテスト
- 初期版はバックエンドなし
- 保存は localStorage から開始してよいが、設計上は IndexedDB に拡張可能にする

既存プロジェクトが別スタックの場合は、既存構成を尊重しながら上記要件を満たす。新規作成なら上記スタックを採用する。

## UI品質基準

- スマホ画面を第一対象にする。
- 1画面1目的にする。
- 今日の行動が3秒以内に理解できる。
- ボタン文言は抽象語ではなく、行動語にする。
- 試験前に疲れているユーザーでも使えるよう、認知負荷を低くする。
- ストリークを強調しすぎず、復帰しやすい設計にする。
- トレーニングアプリ特有の威圧感を避け、落ち着いた、清潔で、継続しやすいUIにする。

## 安全性・健康上の注意

- 医療診断、治療、怪我の診断はしない。
- 痛み、めまい、吐き気、胸痛、強い違和感がある場合は中止する表示を入れる。
- トレーニング強度は「あと2〜3回できる余裕を残す」を基本にする。
- 試験前・睡眠不足時は軽量版または休息を推奨する。

## 実装プロセス

大きな変更を始める前に、必ず実装計画を短く提示する。実装後は次を実行する。

1. 型チェック
2. リント
3. ユニットテスト
4. 主要画面の手動確認、可能ならE2E確認
5. 変更点と残課題の報告

## 完了条件

`docs/09_testing_acceptance.md` の受け入れ条件を満たすこと。特に、次の画面は必須。

- 初期設定画面
- 今日のダッシュボード
- トレーニング実行画面
- 記録・カレンダー画面
- プロテイン/栄養画面
- 設定画面

## 禁止事項

- ユーザーの健康情報を外部送信する実装を勝手に追加しない。
- バックエンド、認証、クラウド同期をMVPに勝手に追加しない。
- 依存関係を過剰に増やさない。
- 完璧な筋肥大アプリに寄せすぎない。7月は試験月であり、目的は習慣化である。
